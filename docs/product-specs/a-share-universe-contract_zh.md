# 产品规格：A 股股票池契约

- 版本：v1
- 日期：2026-04-03
- 适用范围：`model_function/universe.py`、`scripts/filter.py`、`runtime/adapters/modeling.py`、`scripts/build_portfolio.py`

## 1. 目的

本文冻结低算力 A 股模型流程中股票池的 incumbent 行为与重构后契约，使训练、预测和组合阶段都能围绕同一套可验证策略表面进行比较与演进。

## 2. 共用术语

- 成交额代理：优先使用 `amount`，否则退化为 `close * volume`。
- 滞后规则：任何月度或日度成员资格判断，只能使用该时点及之前可见的数据。
- 指数剔除：`index_code_list` 中的符号永不入池。
- ST 剔除：在对应源快照上处于 ST 状态的符号永不入池。

## 3. 既有契约

### 训练股票池

- 按月刷新，采用一个月滞后。
- 排名依据是季度回看窗口上的流动性稳定度，并叠加波动率极值过滤。
- 最终输出通过分组加权选择，写入 `my_800_stocks.txt`。
- 既有训练成员没有显式的月度入池/留任缓冲区。

### 预测股票池

- 先根据最近一段流动性窗口排序。
- 再从高流动性股票中随机抽样出更小的候选池。
- 如果随机池不足目标规模，则用前一交易日预测结果补齐到 500 只。
- 这会让预测资格带有随机性，并把换手控制混入股票池构建阶段。

### 持仓 / 换手控制

- 之前没有显式的模型分数持有带。
- 换手平滑主要依赖前一交易日预测结果的合并逻辑和下游 `top_k` 容量裁剪间接实现。

## 4. 重构后契约

### 训练股票池

- 源周期：前一个自然月。
- 流动性分数：
  - `lagged_liquidity(symbol, month_t) = month_t` 月最后一个可用交易日为止、向前回看 60 个交易观测的 `turnover` 中位数。
  - 最少观测数：20。
- 成员带宽：
  - 入池带：按 `lagged_liquidity` 取前 1800
  - 留任带：按 `lagged_liquidity` 取前 2200
- 留任规则：
  - 新名字只能从入池带进入
  - 旧训练成员只有在仍位于留任带内时才可继续保留
- 算力上限：
  - 最终月度输出会确定性降采样到 800 只
  - 降采样按分段方式执行，且由 `source_month` 与配置 seed 共同保证可复现

### 预测股票池

- 源快照：预测日当天，向前回看截至该日的 60 个交易观测。
- 流动性分数：
  - 按股票计算该 60 日窗口上的成交额中位数
- 成员带宽：
  - 入池带：按滞后流动性取前 1000
  - 留任带：按滞后流动性取前 1200
- 留任规则：
  - 所有入池带内股票都会被打分
  - 前一日 `target_weights_<prev_date>.csv` 中的已有持仓，只要仍位于留任带内，也会继续被打分
- 确定性规则：
  - 预测股票池构建中不允许再使用随机抽样

### 持仓缓冲 / 组合选择

- 新买入带：模型分数前 300。
- 持有带：上一期目标组合中已有股票，只要模型分数仍在前 500，就允许继续保留。
- 最终容量：
  - 先应用买入/持有带，再应用既有 `top_k` 组合容量上限
  - 默认 `top_k` 仍为 80，除非通过 CLI 或 runtime 参数覆盖

## 5. 输出表面

- 训练股票池产物：`.data/qlib_data/instruments/my_800_stocks.txt`
- 预测产物：`output/top_picks_<date>.csv`
- 组合产物：
  - `output/target_weights_<date>.csv`
  - `output/orders_<date>.csv`

## 6. 验收表面

满足以下条件即视为契约落地：

1. 训练股票池成员资格具备月度滞后、确定性、指数/ST 安全和 800 只上限。
2. 相同快照与配置下，预测股票池成员资格完全确定。
3. 持仓留任通过显式买入/持有分数带表达，而不是依赖前一交易日预测结果合并。
4. 训练、预测和组合阶段都通过 `model_function/universe.py` 中的共享 helper 落地。
