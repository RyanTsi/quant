# 执行计划：架构评审

## 目标

评审当前项目结构与软件架构，对照文档中声明的 runtime-first 设计与实际实现，整理一份有依据的优点、问题和高杠杆改进方向总结。

## 范围

- `AGENTS.md` 要求的文档入口与导航内容
- 下列代表性模块：
  - `main.py`
  - `runtime/*`
  - `runtime/adapters/*`
  - `model_function/*`
  - `data_pipeline/*`
  - `alpha_models/*`
  - `scripts/*`
  - `backtesting/*`
  - `test/test_*.py`
- 本次评审的追踪材料：`docs/exec-plans/*` 与 `docs/logs/*`

## 假设

- 用户需要的是架构评审，而不是生产行为修改。
- 当旧文档或历史命名残留与当前实现不一致时，以当前代码与通过的测试为准。
- 当前工作树可能包含进行中的改造，因此结论需要区分稳定的架构信号与重构中的噪声。

## 步骤

1. 按要求顺序阅读导航与核心架构文档。
2. 检查 runtime、数据、模型、CLI 与测试层的代表性模块，梳理真实依赖方向与分层情况。
3. 运行一组聚焦架构表面的测试，验证主要 runtime 与 CLI 接口。
4. 记录双语追踪材料，并向用户总结优点、问题与优先改进建议。

## 验收标准

- 评审基于当前仓库实现，而不只是设计意图文档。
- 优点与问题均对应到具体架构面。
- 已记录聚焦验证结果。
- 双语追踪材料已写入 `docs/exec-plans/*` 与 `docs/logs/*`。

## 回滚说明

- 本任务仅新增文档；若无需保留，可删除新增追踪文件。
